# Submission Checklist — Final QA Before Upload

## Proposal Document

- [ ] **Format:** 2 pages maximum (check page count)
- [ ] **Page 1:** Header, Problem, Solution, Impact — all sections present
- [ ] **Page 2:** Tech Stack, Workflow Diagram, Innovation & Feasibility, Self-Assessment — all present
- [ ] **Track:** Clearly marked as Track 1: Phenome
- [ ] **Project Name:** Present and catchy
- [ ] **Slogan:** Present and meaningful
- [ ] **Success Metrics:** Quantitative metrics included (not vague)
- [ ] **Workflow Diagram:** Visual, clear, present on Page 2
- [ ] **Self-Assessment Table:** Filled in with scores AND justifications
- [ ] **Total Score:** Adds up correctly
- [ ] **Grammar/Spelling:** Proofread
- [ ] **Formatting:** Clean, professional, readable

## POC Prototype

- [ ] **Runs without errors** on target machine (RTX 4060 or Colab)
- [ ] **Dependencies documented** (requirements.txt or equivalent)
- [ ] **Demo data included** (synthetic if real data not available)
- [ ] **Input → Output flow** works end-to-end
- [ ] **Screenshots/recording** captured for proposal inclusion
- [ ] **Non-expert friendly** — medical personnel can understand the output

## Cross-Reference Validation

- [ ] Claims in proposal match what the POC can actually do
- [ ] Neural ODE framed as "novel proposed approach" not "proven technique"
- [ ] Literature references are accurate (check year, author, venue)
- [ ] No contradictions between Page 1 and Page 2
- [ ] Success metrics are achievable by the POC

## File Compliance

- [ ] All files in required format (PDF, PPTX, or as specified)
- [ ] File names match submission requirements
- [ ] File sizes within limits (if specified)
- [ ] All deliverables bundled correctly
