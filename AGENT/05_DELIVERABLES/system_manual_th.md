## 1. ภาพรวมของระบบ

ระบบ **Automated AI Pipeline for NMR Spectroscopy** คือระบบประมวลผลอัตโนมัติแบบ End-to-End ที่ออกแบบมาเพื่อรับข้อมูลสเปกตรัม ¹H NMR ดิบจากเครื่องวัดในห้องปฏิบัติการ (ขนาด 20,000+ ฟีเจอร์ต่อตัวอย่าง) แล้วประมวลผลผ่านโมเดล Deep Learning 3 ชั้น เพื่อ:

- **ลดมิติข้อมูล** จาก 20,000 เหลือ 128 มิติ
- **แก้ไขการเลื่อนตำแหน่งพีค** (Chemical Shift Drift) อัตโนมัติ
- **ตรวจจับและกำจัดพีคหลอก** (Ghost Peak) ที่ผิดกฎฟิสิกส์เคมี
- **ระบุชื่อสารประกอบ** โดยเชื่อมต่อกับฐานข้อมูลระดับโลก (HMDB, PubChem)
- **ส่งออกรายงาน JSON มาตรฐาน** ที่ระบบโรงพยาบาลสามารถนำไปใช้ได้ทันที

```
┌────────────────────────────────────────────────────────────────────────┐
│  ข้อมูลดิบจากเครื่อง NMR (20,000+ ฟีเจอร์)                           │
└──────────────────────────┬─────────────────────────────────────────────┘
                           ▼
              ┌────────────────────────┐
              │  Stage 1: Encoder      │  ลดมิติ 20,000 → 128
              │  (Feature Selection)   │
              └────────────┬───────────┘
                           ▼
              ┌────────────────────────┐
              │  Stage 2: Neural ODE   │  จัดแนวพีคที่ขยับ
              │  (Pattern Recognition) │
              └────────────┬───────────┘
                           ▼
              ┌────────────────────────┐
              │  Stage 3: EBM          │  ตรวจสอบกฎฟิสิกส์ / กำจัดพีคหลอก
              │  (Classification)      │
              └────────────┬───────────┘
                           ▼
              ┌────────────────────────┐
              │  Knowledge Discovery   │  ค้นหาชื่อสารจาก HMDB/PubChem
              └────────────┬───────────┘
                           ▼
              ┌────────────────────────┐
              │  JSON Clinical Report  │  ส่งออกข้อมูลสู่ระบบโรงพยาบาล
              └────────────────────────┘
```

## 2. ปัญหาที่ระบบแก้ไข

### 2.1 ปัญหาของเครื่องมือวิเคราะห์ NMR ในปัจจุบัน

เครื่องมือวิเคราะห์สัญญาณ NMR แบบดั้งเดิม เช่น ASICS, MetaboAnalyst หรือ Chenomx มีข้อจำกัดสำคัญหลายประการ:

| ปัญหา | รายละเอียด | ผลกระทบ |
|-------|-----------|---------|
| **ฟีเจอร์มิติสูง** | สเปกตรัม NMR มีมากกว่า 20,000 จุดข้อมูลต่อตัวอย่าง | อัลกอริทึมแบบเดิม (PCA, LASSO) ทำงานช้าและเลือกฟีเจอร์ผิดพลาด |
| **พีคเลื่อนตำแหน่ง** | อุณหภูมิ ค่า pH และสภาพแวดล้อมทำให้ตำแหน่ง ppm ขยับ | โมเดล ML แบบเดิมมองสัญญาณที่ขยับเป็นฟีเจอร์ใหม่ → จำแนกผิด |
| **พีคหลอก (Ghost Peak)** | ระบบ Statistical Optimization ฝืน Fit สัญญาณนอยส์ | เกิดการระบุสารผิดตัว (Miss-identification) |
| **คลังข้อมูลจำกัด** | ค้นหาสารได้แค่ในคลังท้องถิ่น | พลาดสารบ่งชี้โรคที่อยู่นอกคลัง |
| **ต้องใช้ผู้เชี่ยวชาญ** | ต้องมีนักวิทยาศาสตร์เตรียมข้อมูลทีละไฟล์ | ไม่รองรับการคัดกรองแบบอัตโนมัติความเร็วสูง |

### 2.2 แนวทางการแก้ปัญหาของระบบนี้

ระบบของเราใช้สถาปัตยกรรม **Hybrid AI** ที่รวม 3 เทคนิคล้ำสมัยเข้าด้วยกัน:

1. **Neural Feature Encoder** → แก้ปัญหาฟีเจอร์มิติสูง
2. **Latent Neural ODE** → แก้ปัญหาพีคเลื่อนตำแหน่ง (นวัตกรรมใหม่ — ยังไม่มีงานวิจัยใดทำมาก่อน)
3. **Energy-Based Model** → แก้ปัญหาพีคหลอกและการจำแนกผิด

## 3. สถาปัตยกรรมแบบ 3 ชั้น

ระบบใช้สถาปัตยกรรมแบบ **Dual-Track** ประกอบด้วย:

- **Core Layer (Baseline):** ใช้เทคนิค ML คลาสสิก (PLS-DA + SVM + Random Forest) เพื่อรับประกันว่าระบบทำงานได้อย่างแน่นอน
- **Advanced Layer (Innovation):** ใช้เทคนิค Deep Learning ขั้นสูง (Neural ODE + EBM) เพื่อแสดงนวัตกรรม

ทั้งสองเลเยอร์ทำงานคู่ขนานกัน โดย Core Layer จะทำงานได้เสมอแม้ Advanced Layer ยังอยู่ในขั้นทดลอง

## 4. Stage 1: Neural Feature Encoder

### 4.1 คืออะไร?

**Neural Feature Encoder** คือโมดูลโครงข่ายประสาทเทียม (Neural Network) ที่ทำหน้าที่บีบอัดข้อมูลสเปกตรัม NMR ดิบจาก **20,000 มิติ** ลงเหลือเพียง **128 มิติ** ในพื้นที่มิติแฝง (Latent Space)

### 4.2 ทำอะไร?

- รับเวกเตอร์ความเข้มสเปกตรัมขนาด `[Batch_Size, 20000]`
- บีบอัดผ่าน Linear Layer ขนาด 20,000 → 512 → 128
- ใช้ BatchNorm1d เพื่อปรับสมดุลค่าเอาต์พุต
- ใช้ ReLU Activation เพื่อสร้าง Non-linearity
- ส่งออกเวกเตอร์แฝงขนาด `[Batch_Size, 128]`

### 4.3 ทำงานอย่างไร?

```
ข้อมูลดิบ [N, 20000]
    │
    ▼
┌──────────────────────┐
│  Linear(20000, 512)  │  ← เลเยอร์เชิงเส้นลดมิติครั้งแรก
│  BatchNorm1d(512)    │  ← ปรับสมดุลการกระจายค่า
│  ReLU()              │  ← ฟังก์ชันกระตุ้นแบบไม่เชิงเส้น
│  Linear(512, 128)    │  ← ลดมิติลงสู่พื้นที่แฝง 128 มิติ
└──────────┬───────────┘
           ▼
   เวกเตอร์แฝง [N, 128]
```

**หลักการทำงาน:** แทนที่จะใช้ PCA แบบเดิมที่มองจุดข้อมูลแยกจากกัน Neural Encoder จะ **เรียนรู้** ว่าฟีเจอร์ใดมีความสัมพันธ์กัน (เช่น พีคย่อยของโมเลกุลเดียวกัน) แล้วรวมข้อมูลเหล่านั้นเข้าด้วยกันในมิติแฝง ทำให้การลดมิติไม่สูญเสียความสัมพันธ์เชิงโครงสร้างทางเคมี

### 4.4 แก้ปัญหาอะไร?

| ปัญหาเดิม | วิธีแก้ของ Stage 1 |
|-----------|-------------------|
| PCA/LASSO มองจุดข้อมูลแยกจากกัน | Neural Encoder รักษาความสัมพันธ์ระหว่างพีคย่อย |
| การลดมิติแบบเดิมตัดพีคสำคัญทิ้ง | Encoder เรียนรู้ว่าพีคไหนสำคัญโดยอัตโนมัติ |
| ข้อมูล 20,000 มิติ → โมเดลช้า | บีบอัดเหลือ 128 มิติ (ลดลง **156 เท่า**) |

### 4.5 ตอบเกณฑ์ข้อไหน?

> **เกณฑ์ที่ 1: Feature Selection** — การเลือกตำแหน่งสัญญาณที่สำคัญที่สุด / ลดมิติข้อมูล

## 5. Stage 2: Latent Neural ODE

### 5.1 คืออะไร?

**Latent Neural ODE** (Latent Ordinary Differential Equation Solver) คือโมดูลที่ใช้สมการอนุพันธ์ต่อเนื่อง (Continuous ODE) ในการจำลองและแก้ไขการเลื่อนตำแหน่งพีค (Chemical Shift Drift) ในพื้นที่มิติแฝง

> ⚠️ **หมายเหตุสำคัญ:** นี่คือ **นวัตกรรมใหม่** — ยังไม่มีงานวิจัยใดในโลกที่ประยุกต์ใช้ Neural ODE กับการจัดแนวสัญญาณ NMR มาก่อน

### 5.2 ทำอะไร?

- รับเวกเตอร์แฝงขนาด `[Batch_Size, 128]` จาก Stage 1
- คำนวณอนุพันธ์ (Derivative) ของสถานะปัจจุบันผ่านโครงข่ายประสาท `f_θ`
- อัปเดตสถานะแบบต่อเนื่องด้วย Euler Integration จำนวน 4 ขั้นตอน
- ส่งออกเวกเตอร์แฝงที่ "จัดแนว" แล้ว พร้อมบันทึกเส้นวิถี (Trajectory) ทุกขั้นตอน

### 5.3 ทำงานอย่างไร?

สมการหลักของ Neural ODE:

```
dh(t)/dt = f_θ(h(t), t)
```

โดยที่:
- `h(t)` = สถานะของเวกเตอร์แฝง ณ ขั้นตอน t
- `f_θ` = โครงข่ายประสาทที่กำหนดทิศทางการเปลี่ยนแปลง
- `θ` = พารามิเตอร์ที่เรียนรู้ได้

**กระบวนการ Euler Integration:**

```
ขั้นตอนที่ 0: h₀ = เวกเตอร์แฝงจาก Encoder
ขั้นตอนที่ 1: h₁ = h₀ + 0.1 × f_θ(h₀)
ขั้นตอนที่ 2: h₂ = h₁ + 0.1 × f_θ(h₁)
ขั้นตอนที่ 3: h₃ = h₂ + 0.1 × f_θ(h₂)
ขั้นตอนที่ 4: h₄ = h₃ + 0.1 × f_θ(h₃)  ← เวกเตอร์ที่จัดแนวแล้ว
```

**เหตุผลทางทฤษฎีที่ควรได้ผล:**
1. Chemical Shift Drift เป็นปรากฏการณ์ **ต่อเนื่องและราบเรียบ** — เหมาะกับ ODE
2. Neural ODE ได้รับการพิสูจน์แล้วว่าใช้ได้กับอนุกรมเวลาที่สุ่มตัวอย่างไม่สม่ำเสมอ (Rubanova et al., 2019)
3. สเปกตรัม NMR ตามแกน ppm มีคุณสมบัติทางคณิตศาสตร์คล้ายกับอนุกรมเวลา

### 5.4 แก้ปัญหาอะไร?

| ปัญหาเดิม | วิธีแก้ของ Stage 2 |
|-----------|-------------------|
| โมเดล ML แบบเดิมมองพีคที่ขยับเป็นฟีเจอร์ใหม่ | Neural ODE จัดแนวพีคกลับตำแหน่งอ้างอิงอัตโนมัติ |
| วิธี Binning สูญเสียความละเอียดของสัญญาณ | ODE ทำงานในมิติแฝงโดยไม่ต้องลดความละเอียด |
| ต้องจัดแนวด้วยมือ (Manual Alignment) | จัดแนวอัตโนมัติ 100% ไม่ต้องใช้ผู้เชี่ยวชาญ |

### 5.5 ตอบเกณฑ์ข้อไหน?

## 6. Stage 3: Energy-Based Model (EBM)

### 6.1 คืออะไร?

**Energy-Based Model (EBM)** คือโมดูลที่คำนวณ "ค่าพลังงาน" (Energy Score) ของโครงสร้างสัญญาณแต่ละตัวอย่าง โดยอิงจากกฎเกณฑ์ทางฟิสิกส์เคมี เช่น อัตราส่วนความสูงพีค (Spin-Spin Coupling) และรูปแบบ Multiplet

### 6.2 ทำอะไร?

- รับเวกเตอร์แฝงที่จัดแนวแล้วจาก Stage 2
- คำนวณ **Energy Score** แบบ Scalar (ค่าเดียว) สำหรับแต่ละตัวอย่าง
- เปรียบเทียบ Energy Score กับ **Threshold** (ค่าขีดจำกัด)
- ถ้า Energy > Threshold → **ตรวจพบ Ghost Peak** → กดค่าพลังงานลงและตั้งค่าสถานะแจ้งเตือน
- ถ้า Energy ≤ Threshold → **ผ่านการตรวจสอบ** → สัญญาณปลอดภัย

### 6.3 ทำงานอย่างไร?

```
เวกเตอร์แฝงที่จัดแนว [N, 128]
    │
    ▼
┌──────────────────────┐
│  Linear(128, 32)     │  ← ลดมิติเพื่อประมาณค่าพลังงาน
│  ReLU()              │
│  Linear(32, 1)       │  ← ส่งออก Energy Score (ค่าเดียว)
└──────────┬───────────┘
           ▼
    Energy Score (สเกลาร์)
           │
           ▼
   ┌───────────────────┐
   │  Score > Threshold │──── ใช่ ──→  ⚠️ Ghost Peak Detected!
   │      ?            │                   → กดค่าพลังงานลง
   └───────┬───────────┘                   → แจ้งเตือนผู้ใช้
           │
         ไม่ใช่
           │
           ▼
     ✅ Physics Verified
        สัญญาณปลอดภัย
```

**หลักการฟิสิกส์เคมีที่ EBM บังคับใช้:**
- **กฎ Multiplet:** สารแต่ละชนิดมีรูปแบบ Singlet/Doublet/Triplet ที่แน่นอน
- **ค่าคงที่ J-coupling:** ค่าคงที่สำหรับพันธะแต่ละประเภท
- **อัตราส่วนความสูงพีค:** ต้องเป็นไปตาม Binomial Distribution สำหรับ Coupled Spins
- **ช่วง Chemical Shift:** สารแต่ละกลุ่มปรากฏในช่วง ppm ที่ทราบแล้ว

### 6.4 แก้ปัญหาอะไร?

| ปัญหาเดิม | วิธีแก้ของ Stage 3 |
|-----------|-------------------|
| ASICS ฝืน Fit สัญญาณนอยส์จนเกิด Ghost Peak | EBM ตรวจจับ Ghost Peak ผ่านค่าพลังงานที่พุ่งสูง |
| ไม่มีระบบตรวจสอบความถูกต้องทางวิทยาศาสตร์ | EBM บังคับให้คำตอบเป็นไปตามกฎฟิสิกส์เคมี |
| ระบุสารผิดตัว (Miss-identification) | ค่าพลังงานสูง = สัญญาณไม่น่าเชื่อถือ → ปฏิเสธอัตโนมัติ |

### 6.5 ตอบเกณฑ์ข้อไหน?

> **เกณฑ์ที่ 3: Compound Classification** — การจำแนกประเภทสารและการกำจัด Ghost Peak

## 7. ระบบค้นหาสารบ่งชี้โรคอัตโนมัติ

### 7.1 คืออะไร?

**Automated Knowledge Discovery** คือโมดูลที่จัดกลุ่มสัญญาณปริศนาที่ไม่มีชื่อในคลังข้อมูลท้องถิ่น แล้วส่งคำสั่ง API ไปตรวจสอบกับฐานข้อมูลระดับโลกโดยอัตโนมัติ

### 7.2 ทำอะไร?

- รับผลการจำแนกจาก Stage 3
- จัดกลุ่มพิกัดสัญญาณปริศนาที่ขยับตัวไปด้วยกัน (Non-annotated Features)
- ส่งคำสั่ง API Query ไปยัง:
  - **HMDB** (Human Metabolome Database) — ฐานข้อมูลเมตาบอไลต์ของมนุษย์
  - **PubChem** — ฐานข้อมูลสารเคมีระดับโลกของ NIH
- ส่งคืนข้อมูลชื่อสาร สูตรเคมี และค่าความเชื่อมั่น (Match Confidence)

### 7.3 ตัวอย่างผลลัพธ์

```json
{
  "predicted_extract": "Plant_Extract_A",
  "screened_biomarkers": [
    {

      "match_confidence": 0.96,
      "formula": "C₁₆H₁₈O₉"
    },
    {
      "compound_name": "Formate",
      "source_database": "HMDB0000142",
      "match_confidence": 0.91,
      "formula": "CH₂O₂"
    }
  ]
}
```

### 10.2 กระบวนการ

```

    │

    │
    ▼
Water Region Removal (ลบสัญญาณน้ำ 4.5-5.0 ppm)

    ▼

│ F1 ≈ 0.83   │    │ F1 ≈ 0.81       │
└─────────────┘    └──────────────────┘
```

## 11. ตัวจำลองข้อมูลสังเคราะห์

### 11.1 คืออะไร?

**Synthetic NMR Generator** คือโมดูลที่สร้างสเปกตรัม NMR จำลองแบบสมจริง สำหรับใช้ทดสอบระบบก่อนที่จะได้รับข้อมูลจริงจากสถาบัน

### 11.2 คุณสมบัติ

- สร้างสเปกตรัมขนาด 20,000 จุดข้อมูลสำหรับ 3 คลาสสารประกอบ
- จำลอง **Lorentzian Peak Profile** ตามทฤษฎี NMR
- จำลอง **Chemical Shift Drift** (ปรับได้ผ่านพารามิเตอร์)
- จำลอง **Baseline Gaussian Noise** (ปรับได้ผ่านพารามิเตอร์)
- จำลอง **Ghost Peak Injection** (เปิด/ปิดได้)

### 11.3 คลาสสารประกอบที่จำลอง

| คลาส | ลักษณะ | ตำแหน่งพีคหลัก (ppm) |
|-------|--------|---------------------|
| Plant Extract A | น้ำตาล (Glucose-like) | 1.33, 2.08, 3.24, 5.23 |
| Plant Extract B | สารอะโรมาติก (Flavonoid-like) | 1.18, 2.35, 6.25, 7.45 |
| Plant Extract C | เทอร์ปีน (Terpene-like) | 0.88, 1.62, 3.30, 5.10 |

## 12. สรุปการตอบโจทย์เกณฑ์การประเมิน

ตารางสรุปว่าแต่ละส่วนของระบบตอบเกณฑ์การประเมินข้อใดบ้าง:

| เกณฑ์ | คำอธิบาย | โมดูลที่ตอบโจทย์ | วิธีแก้ |
|-------|----------|-----------------|--------|
| **1. Feature Selection** | ลดมิติข้อมูลจาก 20,000+ | Stage 1: Neural Feature Encoder | บีบอัด 20,000 → 128 มิติ (156x) โดยรักษาความสัมพันธ์เชิงโครงสร้าง |
| **2. Pattern Recognition** | ระบุรูปแบบสัญญาณ | Stage 2: Latent Neural ODE | จัดแนวพีคที่ขยับด้วยสมการอนุพันธ์ต่อเนื่อง (นวัตกรรมใหม่) |
| **3. Classification** | จำแนกสาร + กำจัด Ghost Peak | Stage 3: Energy-Based Model | คำนวณค่าพลังงาน → ตรวจจับ Ghost Peak → กดทิ้งอัตโนมัติ |
| **4. Biomarker Discovery** | ค้นหาสารบ่งชี้โรค | Automated Knowledge Discovery | ส่ง API Query ไปยัง HMDB/PubChem → ระบุชื่อสารอัตโนมัติ |
| **5. Workflow Development** | Pipeline อัตโนมัติ TRL 4/5 | Clinical JSON Report + Dashboard | Structured JSON + Streamlit Dashboard → ใช้งานได้ทันที |

## คำศัพท์เทคนิค

| คำศัพท์ | ความหมาย |
|---------|---------|
| **NMR** | Nuclear Magnetic Resonance — เทคนิคการวัดสนามแม่เหล็กของนิวเคลียส |
| **ppm** | Parts Per Million — หน่วยวัดตำแหน่ง Chemical Shift |
| **Latent Space** | พื้นที่มิติแฝง — การแสดงข้อมูลในมิติที่ต่ำกว่า |
| **ODE** | Ordinary Differential Equation — สมการอนุพันธ์สามัญ |
| **EBM** | Energy-Based Model — โมเดลที่ใช้ฟังก์ชันพลังงาน |
| **Ghost Peak** | พีคหลอกที่เกิดจากการ Fit ข้อมูลผิดพลาด |
| **TRL** | Technology Readiness Level — ระดับความพร้อมของเทคโนโลยี |
| **HMDB** | Human Metabolome Database — ฐานข้อมูลเมตาบอไลต์ |
| **Lorentzian** | รูปทรงพีคตามทฤษฎี NMR |
| **VIP Score** | Variable Importance in Projection — ค่าความสำคัญของตัวแปรใน PLS-DA |
| **BatchNorm** | Batch Normalization — เทคนิคปรับสมดุลเอาต์พุตระหว่างเลเยอร์ |
| **Euler Integration** | วิธีการประมาณคำตอบ ODE แบบทีละขั้น |
