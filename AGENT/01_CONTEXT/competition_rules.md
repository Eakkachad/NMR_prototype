# Competition Rules — BDI YOUNG INNOVATOR HACKATHON 2026

> Source: `proposalform.md` and `OrientationDay-hackathon-datasetV3-BI8PKue_.pptx`

## Event Identity

- **Name:** BDI Young Innovator Hackathon 2026
- **Track:** Track 1 — Phenome (Metabolomics, Compound Profiling)
- **Organizer:** Not explicitly stated; affiliated with National Phenome Institute
- **Duration:** 7 days

## Submission Requirements

### 1. Written Proposal (2 pages)

**Page 1 — The Vision & Impact:**
| Section | Requirement |
|---------|-------------|
| Header | Track selection, Project name, Slogan, Target audience |
| Problem Statement | Core challenge, significance, current gap |
| Solution Approach | High-level summary of proposed solution |
| Expected Impact | Hackathon impact, 12-24 month impact, **quantitative success metrics** |

**Page 2 — The Execution & Evaluation:**
| Section | Requirement |
|---------|-------------|
| Tech Stack | AI/ML models, databases, pipelines, platform, deployment, security |
| Workflow Diagram | Visual flowchart from data input to final UI |
| Innovation & Feasibility | What's novel, why it's buildable, risks |
| Self-Assessment | Score yourself against each rubric criterion with justification |

### 2. Prototype / POC
- Must demonstrate feasibility within 2-month window
- Can be minimal — this is explicitly a POC

## Scoring Rubric (100 points)

| Criterion | Points | What Judges Ask |
|-----------|--------|-----------------|
| **Feasibility** | 30 | Can you build a Prototype/PoC within 2 months? Show team technical capacity |
| **Problem Match & Dataset** | 25 | Does project perfectly match the problem? How will data be sourced/handled? |
| **Sustainable Impact** | 20 | Long-term, equitable change? |
| **Novelty** | 15 | Is this genuinely new, not a copy? |
| **Implementation Plan** | 10 | Concrete plan to execute within hackathon? |

## Constraints & Rules

- Track choice is binding (we are Track 1: Phenome)
- Dataset comes from National Phenome Institute
- Solution should be deliverable (web app, mobile app, dashboard, or API)
- Security considerations expected (encryption, secure data handling)
- Self-assessment section is REQUIRED — you must justify your own score

## Strategic Notes

> [!TIP]
> **Feasibility is 30% of the score** — the largest single criterion. A working POC demo, even simple, is more valuable than a complex theoretical proposal that can't be demonstrated.

> [!IMPORTANT]
> **Success Metrics are mandatory.** The proposal must include quantitative metrics (e.g., "classify compounds with >85% accuracy", "reduce interpretation time from 4 hours to 5 minutes").
